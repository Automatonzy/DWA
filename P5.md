# P5: Flat Arm + Locomotion Merge

## 1. 定位

`P5` 是从 `P4` 的站立动臂，过渡到平地上的低速腿臂融合阶段。

它的目标不是 rough，也不是接触任务，而是：

- 保留 `P4` 的大范围 arm 控制
- 重新引入底盘低速移动
- 让机器人做到 `30%` 站立、`70%` 运动 的混合训练

对应任务：

- `RobotLab-Isaac-Velocity-Flat-Go2-X5-ArmLocomotion-v0`

## 2. 起点模型

`P5` 直接承接 `P4` checkpoint。

推荐优先顺序：

1. `P4` 训练过程中挑出来的 best checkpoint
2. 如果没有单独筛选 best，再考虑 `500` 轮附近的 checkpoint
3. 最后才是 `8500.pt`

如果你现在只是先把链路跑通，直接从 `P4 8500.pt` 开始也可以。

## 3. 输入输出维度

`P5` 继续沿用 route 接口，不改网络结构：

- actor obs: `259`
- critic obs: `259`
- action: `18`

所以它可以直接 `resume`，不需要迁移 checkpoint。

## 4. 环境语义

`P5` 仍然是 flat terrain，但不再是纯站立任务。

- terrain: flat
- `num_envs = 2048`
- standing ratio: `0.30`
- moving ratio: `0.70`
- base command:
  - `lin_vel_x = (-0.15, 0.15)`
  - `lin_vel_y = (-0.10, 0.10)`
  - `ang_vel_z = (-0.18, 0.18)`
- base command 不做 curriculum，固定保持 `30%` 站立 / `70%` 运动

## 5. Arm Command

`P5` 不再对机械臂做课程学习。

也就是说：

- arm command 从一开始就是 full-range
- 直接继承 `P4` 的最终 command 支持范围
- 不再做 arm curriculum

当前使用的 arm command range：

- `joint1: (-2.40, 3.00)`
- `joint2: (0.00, 3.00)`
- `joint3: (0.00, 3.00)`
- `joint4: (-1.45, 1.45)`
- `joint5: (-1.45, 1.45)`
- `joint6: (-1.45, 1.45)`

要注意的硬限制：

- `joint2` 和 `joint3` 不能为负
- `joint1 / joint4 / joint5 / joint6` 可以为负

arm action scale：

- `joint1: 1.20`
- `joint2: 1.20`
- `joint3: 1.20`
- `joint4: 0.80`
- `joint5: 0.70`
- `joint6: 0.70`

## 6. 奖励设计

`P5` 不是从零开始配 reward，而是从 `P4` 权重平滑过渡到 `P5 target`。

reward curriculum:

- `curriculum_iterations = 1000`

### 6.1 P5 target reward weights

- `track_lin_vel_xy_exp = 4.5`
- `track_ang_vel_z_exp = 2.2`
- `lin_vel_z_l2 = -1.6`
- `ang_vel_xy_l2 = -0.10`
- `flat_orientation_l2 = -0.50`
- `base_height_l2 = -0.2`
- `body_lin_acc_l2 = -0.015`
- `joint_torques_l2 = -1.5e-5`
- `joint_acc_l2 = -1.0e-7`
- `joint_pos_limits = -2.0`
- `joint_power = -1.0e-5`
- `stand_still = -2.0`
- `joint_pos_penalty = -0.9`
- `action_rate_l2 = -0.01`
- `undesired_contacts = -1.0`
- `contact_forces = -1.0e-4`
- `feet_air_time = 0.08`
- `feet_air_time_variance = -0.2`
- `feet_contact_without_cmd = 0.2`
- `feet_slide = -0.12`
- `feet_gait = 0.20`
- `arm_joint_pos_tracking_l2 = -3.2`
- `arm_joint_vel_l2 = -0.001`
- `arm_joint_acc_l2 = -7.5e-7`
- `arm_joint_torques_l2 = -6.0e-5`
- `arm_action_rate_l2 = -0.010`
- `arm_joint_pos_limits = -2.0`
- `arm_joint_deviation_l2 = 0.0`
- `arm_motion_tilt_penalty = -0.30`
- `arm_action_in_unstable_base = -0.10`
- `arm_stable_track_bonus = 1e-6`

### 6.2 设计逻辑

- `P4` 的 `stand_still` 很重，因为那是纯站立动臂。
- `P5` 里 `stand_still` 放松到 `-2.0`，因为现在只有 `30%` 样本需要站立。
- `track_lin_vel_xy_exp / track_ang_vel_z_exp` 被抬高，因为底盘重新有命令了。
- `feet_air_time / feet_gait` 重新打开，让策略恢复低速步态组织能力。
- `arm_joint_pos_tracking_l2` 仍然保持中高权重，避免一恢复底盘运动就把 arm 能力忘掉。
- `arm_motion_tilt_penalty / arm_action_in_unstable_base` 继续保留，因为腿臂耦合稳定性仍然是主问题。

## 7. 域随机化

`P5` 仍然保持保守随机化，不让外部干扰盖过“腿臂融合”本身的问题。

- friction:
  - `static = (0.60, 1.05)`
  - `dynamic = (0.50, 0.95)`
  - `restitution = (0.0, 0.10)`
- base mass scale: `(0.95, 1.05)`
- other body mass scale: `(0.97, 1.04)`
- base COM perturbation: `+-0.015 m`
- actuator gains:
  - interval `3.0 ~ 5.0 s`
  - stiffness `(0.93, 1.07)`
  - damping `(0.93, 1.07)`
- external force/torque: disabled
- push: disabled
- obs noise:
  - `base_ang_vel: [-0.02, 0.02]`
  - `projected_gravity: [-0.025, 0.025]`
- action delay:
  - `delay_range = (0, 1)`
  - `hold_prob = 0.03`
  - `noise_std = 0.004`
- observation delay: `0` step

## 8. PPO 超参数

`P5` 是 fine-tune whole-body coordination，不建议继续用 `P4` 的 `5e-4`。

当前实现：

- `num_steps_per_env = 32`

- `save_interval = 250`
- `experiment_name = go2_x5_arm_locomotion_flat`
- actor hidden dims: `[512, 256, 128]`
- critic hidden dims: `[512, 256, 128]`
- `learning_rate = 2e-4`
- `entropy_coef = 0.0025`
- `gamma = 0.99`
- `lam = 0.95`
- `num_learning_epochs = 5`
- `num_mini_batches = 4`
- `desired_kl = 0.01`

## 9. 训练命令

如果你直接从 `P4 8500.pt` 开始：

```bash
python scripts/reinforcement_learning/rsl_rl/train.py \
  --task=RobotLab-Isaac-Velocity-Flat-Go2-X5-ArmLocomotion-v0 \
  --headless \
  --num_envs=2048 \
  --resume \
  --no_load_optimizer \
  --checkpoint=logs/rsl_rl/go2_x5_arm_unlock_flat/2026-03-18_13-33-47/model_8500.pt
```

如果你后面挑出了 `P4` 更好的 checkpoint，把上面的 `--checkpoint` 路径换掉就行。

## 10. 主监控指标

`P5` 不要只看 `stand_still`，也不要只看 `arm_joint_pos_tracking_l2`。

最值得盯的 6 项：

- `track_lin_vel_xy_exp`
- `track_ang_vel_z_exp`
- `arm_joint_pos_tracking_l2`
- `flat_orientation_l2`
- `feet_slide`
- `arm_motion_tilt_penalty`

判断逻辑：

- velocity tracking 变好，但 arm tracking 变差：arm 被忘掉了
- arm tracking 还行，但姿态、脚滑、tilt penalty 变差：腿臂耦合不稳
- velocity tracking 和 arm tracking 都还行，而且姿态稳定：`P5` 成立

## 11. 过关标准

建议至少满足下面这些再进下一阶段：

- `0 0 0` 命令下，机器人能可靠回到稳定站立
- 低速 `x / y / yaw` 命令下，base tracking 不明显发散
- arm full-range command 下，tracking 仍然可用
- 动臂时，底盘不会持续前冲、侧飘或高频摆动
- `feet_slide` 不明显恶化
